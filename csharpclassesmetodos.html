Classes em C#

As classes são o coração da programação orientada a objetos em C#. Elas funcionam como moldes para criar objetos, encapsulando dados (atributos) e comportamentos (métodos) relacionados em uma única unidade.

Imagine uma classe como uma receita para fazer um bolo. A receita (classe) define os ingredientes (atributos) e os passos para fazer o bolo (métodos). Cada bolo que você faz seguindo essa receita é um objeto dessa classe.

Estrutura Básica de uma Classe

public class NomeDaClasse
{
    // Atributos (variáveis da classe)
    public string Atributo1 { get; set; }
    public int Atributo2 { get; set; }

    // Método construtor
    public NomeDaClasse(string valor1, int valor2)
    {
        Atributo1 = valor1;
        Atributo2 = valor2;
    }

    // Métodos
    public void Metodo1()
    {
        // Código do método
    }

    public string Metodo2()
    {
        // Código do método
        return "Algum resultado";
    }
}

Vamos ver um exemplo prático de uma classe Pessoa:

public class Pessoa
{
    public string Nome { get; set; }
    public int Idade { get; set; }

    public Pessoa(string nome, int idade)
    {
        Nome = nome;
        Idade = idade;
    }

    public void Apresentar()
    {
        Console.WriteLine($"Olá, meu nome é {Nome} e eu tenho {Idade} anos.");
    }
}

Neste exemplo, Pessoa é uma classe com dois atributos (Nome e Idade) e um método (Apresentar). O construtor permite criar uma nova Pessoa fornecendo um nome e uma idade.

Métodos Static em C#

Métodos static em C# são métodos que pertencem à classe em si, não a instâncias específicas da classe. Isso significa que você pode chamar um método static sem criar um objeto da classe.

Pense em métodos static como ferramentas utilitárias que não precisam de informações específicas de um objeto para funcionar. Eles são úteis para operações que são independentes do estado do objeto.

Características dos Métodos Static

  • Pertencem à classe, não a instâncias da classe
  • Podem ser chamados sem criar um objeto da classe
  • Não podem acessar membros não-static diretamente
  • São úteis para operações utilitárias e funções auxiliares

Exemplo de Método Static

public class Calculadora
{
    public static int Somar(int a, int b)
    {
        return a + b;
    }

    public static int Multiplicar(int a, int b)
    {
        return a * b;
    }
}

Neste exemplo, Somar e Multiplicar são métodos static. Você pode usá-los assim:

int resultado1 = Calculadora.Somar(5, 3);      // resultado1 será 8
int resultado2 = Calculadora.Multiplicar(4, 2); // resultado2 será 8

Observe que não precisamos criar uma instância de Calculadora para usar esses métodos.

Métodos Void em C#

Métodos void em C# são métodos que executam uma ação, mas não retornam um valor. O termo "void" significa literalmente "vazio" ou "sem valor de retorno".

Pense em métodos void como ações que você realiza sem esperar um resultado específico de volta. Por exemplo, imprimir algo na tela ou salvar dados em um arquivo são ações que não precisam retornar um valor.

Características dos Métodos Void

  • Não retornam um valor
  • Usados para realizar ações ou operações
  • Não é necessário usar a palavra-chave 'return' (embora possa ser usada para sair do método prematuramente)
  • Ideais para métodos que modificam o estado do objeto ou realizam uma operação sem produzir um resultado

Exemplo de Método Void

public class Mensageiro
{
    public void EnviarMensagem(string mensagem)
    {
        Console.WriteLine($"Enviando mensagem: {mensagem}");
        // Imagine que aqui teríamos código para realmente enviar a mensagem
    }

    public void LimparHistorico()
    {
        Console.WriteLine("Limpando o histórico de mensagens...");
        // Aqui teríamos código para limpar o histórico
    }
}

Neste exemplo, tanto EnviarMensagem quanto LimparHistorico são métodos void. Eles realizam ações, mas não retornam valores. Você os usaria assim:

Mensageiro meuMensageiro = new Mensageiro();
meuMensageiro.EnviarMensagem("Olá, mundo!");
meuMensageiro.LimparHistorico();

Modificador Public em C#

O modificador public em C# é um modificador de acesso que define a visibilidade de tipos e membros de tipos. Quando um membro de classe (como um método ou propriedade) é declarado como public, ele pode ser acessado de qualquer lugar, tanto dentro quanto fora da classe.

Pense no modificador public como uma porta aberta. Qualquer um pode entrar e usar o que está lá dentro.

Características do Modificador Public

  • Oferece o nível mais amplo de acesso
  • Membros public podem ser acessados de qualquer parte do código
  • Útil para interfaces e APIs que precisam ser acessíveis externamente
  • Deve ser usado com cuidado para manter o encapsulamento

Exemplo de Uso do Modificador Public

public class Conta
{
    public string Titular { get; set; }
    public decimal Saldo { get; private set; }

    public Conta(string titular, decimal saldoInicial)
    {
        Titular = titular;
        Saldo = saldoInicial;
    }

    public void Depositar(decimal valor)
    {
        if (valor > 0)
        {
            Saldo += valor;
            Console.WriteLine($"Depósito de {valor:C} realizado. Novo saldo: {Saldo:C}");
        }
    }

    public void ExibirSaldo()
    {
        Console.WriteLine($"Saldo atual de {Titular}: {Saldo:C}");
    }
}

Neste exemplo, Titular, Depositar, e ExibirSaldo são membros public. Eles podem ser acessados de fora da classe:

Conta minhaConta = new Conta("João", 1000);
minhaConta.Depositar(500);
minhaConta.ExibirSaldo();
Console.WriteLine($"Titular da conta: {minhaConta.Titular}");

Note que Saldo tem um getter public, mas um setter private, o que significa que pode ser lido de fora da classe, mas só pode ser modificado dentro da classe.
